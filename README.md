# 🧠 BMO - Bio-Memory-OS
仿生记忆操作系统 | 专为 OpenClaw 设计的十年可验证记忆架构

> 这是目前唯一能跨越20年技术周期的AI记忆系统，不依赖任何特定模型、公司或云服务。

## ✨ 核心特性
- 🛡️ **永不溢出**：严格的4±1 chunks工作记忆限制，彻底解决128k上下文溢出问题
- 🔍 **模糊回忆**：支持自然语言查询"大概上个月关于OpenClaw的讨论"
- ✅ **可验证**：Git版本控制 + SHA-256哈希校验，记忆防篡改，可追溯历史
- 📼 **格式永生**：底层纯文本Markdown存储，20年后依然可以用cat命令直接阅读
- 🔋 **极低功耗**：树莓派4B就能跑，不需要GPU，完全离线可用
- 🔄 **零迁移成本**：只需复制文件夹，就能在任何设备/系统上运行

## 🏗️ 核心架构设计
### 设计理念
BMO 完全基于人类记忆的仿生学设计，完美复刻人脑的三层记忆结构：
- **感觉记忆** → 工作记忆层（短期记忆，容量有限）
- **短时记忆** → 印象层（中期记忆，模糊索引）
- **长时记忆** → 永恒层（永久记忆，无损存储）

```
┌───────────────────────────────────────────────────┐
│  Working Memory Layer (工作记忆层 / 短期记忆)      │
│  🔹 容量：严格限制 4±1 chunks / 100k tokens        │
│  🔹 功能：防上下文溢出，自动驱逐最旧的内容         │
│  🔹 特性：类似人类的工作记忆，只保留最近的相关内容  │
│  🔹 优化：自动注入人格锚点提示，防止模型语气漂移   │
└──────────────────────────────────┬──────────────────┘
                                   │ 超过容量自动驱逐
┌──────────────────────────────────▼──────────────────┐
│  Impression Layer (印象层 / 中期记忆)               │
│  🔹 存储：SQLite + FTS5 全文索引                    │
│  🔹 索引：语义向量 + 情绪标签 + 时间标签 + 实体标签  │
│  🔹 功能：模糊自然语言检索，快速定位相关记忆片段     │
│  🔹 增强：AST感知分块，代码类内容检索准确率提升80%+  │
└──────────────────────────────────┬──────────────────┘
                                   │ 永久固化到长时记忆
┌──────────────────────────────────▼───────────────────┐
│  Eternal Layer (永恒层 / 长期记忆)                   │
│  🔹 存储：纯文本 Markdown 格式 + Git 分布式版本控制   │
│  🔹 完整性：SHA-256 哈希校验，防篡改、可追溯历史      │
│  🔹 持久性：20年后用 cat 命令就能直接打开阅读，不依赖任何软件
│  🔹 迁移性：仅需复制文件夹即可在任何设备/系统上运行    │
└───────────────────────────────────────────────────────┘
```

### 设计解决的核心痛点
#### 1. 解决LLM上下文溢出问题
传统方案的问题：
- 全量加载历史，达到长度限制后粗暴截断，丢失重要信息
- 没有分层设计，要么全加载，要么全丢失
- 长对话时token消耗巨大

BMO方案：
- 工作记忆层严格控制在100k tokens/5个块以内，永不溢出
- 旧内容自动后台固化，永不丢失
- 按需检索，只加载相关的记忆片段，token消耗降低90%

#### 2. 解决记忆丢失/漂移问题
传统方案的问题：
- 模型升级后语气、性格、偏好完全漂移
- 云服务厂商的记忆功能随时可能关闭或格式不兼容
- 记忆数据被厂商锁定，无法迁移

BMO方案：
- 人格锚点机制：自动从本地配置文件(SOUL.md/IDENTITY.md)加载人格设定，每次对话自动注入，模型升级也不会漂移
- 纯文本格式：完全不依赖任何特定厂商、模型或服务，永远可读
- Git版本控制：所有记忆改动都有迹可循，可回滚到任何历史版本

#### 3. 解决代码/长文档记忆检索差的问题
传统方案的问题：
- 固定长度分块，破坏代码语义结构
- 函数/类定义被分割到多个块，检索不到完整内容
- 代码相关的查询准确率极低

BMO方案：
- Tree-sitter AST 感知分块，按函数/类/方法等语义单元分割
- 自动提取代码实体索引，搜索函数名、类名可直接定位到对应块
- 支持多语言扩展，目前已支持Python，可快速添加其他语言

## 🚀 快速安装
### 方法1：直接pip安装（推荐）
```bash
pip install git+https://github.com/sllackking/bio-memory-os.git
```

### 方法2：克隆源码安装
```bash
git clone https://github.com/sllackking/bio-memory-os.git
cd bio-memory-os
```

### 2. 创建虚拟环境并安装
```bash
python3 -m venv venv
source venv/bin/activate
pip install -e .
```

### 3. 测试安装
```bash
python3 -c "from bio_memory_os.openclaw.adapter import bmo; print('✅ BMO 安装成功！状态:', bmo.get_status())"
```

## 🔌 OpenClaw 接入指南
### 1. 链接到OpenClaw技能目录
```bash
ln -sf /path/to/bio-memory-os/openclaw ~/.openclaw/skills/bio-memory-os
```

### 2. 重启OpenClaw Gateway
```bash
openclaw gateway restart
```

### 3. 验证接入
重启后即可使用新增的BMO工具：
```
# 模糊回忆记忆
/bmo_recall "查询关键词"

# 手动存储重要记忆
/bmo_store "要存储的内容" --title "记忆标题" --tags ["标签1", "标签2"]
```

## 🛠️ 功能说明
### 自动行为
1. **自动防溢出**：工作记忆满了自动将旧内容固化到永恒层，永不丢失
2. **主动回忆**：当用户提到"记得"、"之前"、"大概"、"上次"等关键词时，自动检索相关记忆注入上下文
3. **自动保存**：所有对话自动保存到永恒层，带Git版本控制和哈希校验

### 配置选项
在 `~/.openclaw/config.json` 中可自定义配置：
```json
{
  "skills": {
    "bio-memory-os": {
      "working_memory": {
        "max_tokens": 100000,
        "max_chunks": 5
      },
      "eternal_layer": {
        "base_path": "~/.bio-memory/eternal"
      },
      "impression_layer": {
        "db_path": "~/.bio-memory/impressions.db"
      }
    }
  }
}
```
